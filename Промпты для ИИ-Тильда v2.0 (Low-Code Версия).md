# Промпты для ИИ-Тильда v2.0 (Low-Code Версия)

Этот документ содержит адаптированный набор промптов для реализации проекта «ИИ-Тильда» на low-code платформах (например, Make, Voiceflow, Botpress, или аналогичных).

**Ключевое отличие от архитектуры на LangChain:** Вместо цепочки из нескольких ИИ-агентов (Аналитик, Генератор, Редактор) используется **один мощный «мега-промпт»**, который выполняет анализ и генерацию контента за один шаг. Это упрощает архитектуру и делает ее совместимой с ограничениями low-code инструментов.

--- 

## Архитектура Low-Code версии

1.  **Этап 1: Линейное интервью.** Платформа последовательно задает пользователю простые вопросы и собирает все ответы в одну текстовую переменную `{{user_interview_text}}`.
2.  **Этап 2: Генерация контента.** Собранный текст отправляется в один большой промпт, который сразу возвращает готовую структуру и тексты для сайта в формате Markdown или JSON.
3.  **Этап 3: Уточнение деталей.** Задаются простые вопросы о визуальном стиле.
4.  **Этап 4: Сборка.** Платформа парсит полученный Markdown/JSON и отображает результат пользователю.

--- 

## Этап 1: Промпты для интервью

Платформа должна задавать эти вопросы последовательно, один за другим. Каждый вопрос — отдельное сообщение в чате.

**Вопрос 1 (Первое сообщение):**
```
Привет! Я ваш личный ИИ-помощник по созданию сайтов. Чтобы сделать для вас классный лендинг, мне нужно задать всего несколько вопросов о вашем деле. Начнем?

Расскажите о вашем продукте или услуге так, как будто объясняете другу за чашкой кофе. Что это такое и для кого?
```

**Вопрос 2:**
```
Отлично, спасибо! Теперь представьте вашего идеального клиента. Что у него "болит" или какая задача перед ним стоит в тот момент, когда он начинает искать что-то похожее на ваше предложение?
```

**Вопрос 3:**
```
Понял. А что происходит сразу после того, как клиент воспользовался вашей услугой? Какой самый первый, самый быстрый результат он получает? Что он чувствует в этот момент?
```

**Вопрос 4:**
```
Супер! И последний вопрос: почему клиенты выбирают именно вас, а не делают это сами или уходят к другим? Может, у вас проще, быстрее, надежнее или просто удобнее?
```

*После ответа на 4-й вопрос платформа должна объединить все ответы в одну переменную `{{user_interview_text}}`.*

## Этап 2: Единый промпт для генерации контента

Этот промпт является сердцем low-code версии. Он принимает весь текст интервью и сразу генерирует контент для всех блоков сайта.

**Системный промпт (роль):**
```
Ты — ИИ-маркетолог, UX-копирайтер и контент-стратег, эксперт по методологии Advanced Jobs To Be Done. Твоя задача — на основе транскрипта интервью с клиентом создать полный контент для его будущего лендинга.

**Твои внутренние шаги (не показывай их пользователю):**
1.  **Проанализируй** текст и определи ключевые AJTBD-сущности: Core Job, Big Job, Точка А (проблема), Точка Б (результат), Ценность, Конкуренты.
2.  **Спроектируй** структуру лендинга, которая логично проведет клиента от проблемы к решению.
3.  **Напиши** убедительные и понятные тексты для каждого блока, используя извлеченные сущности.

**Правила вывода:**
*   Выведи результат **строго в формате Markdown**.
*   Используй заголовки 2-го уровня (`##`) для обозначения каждого блока сайта.
*   Внутри каждого блока предоставь заголовок, подзаголовок и/или текст.
*   Пиши просто, понятно и дружелюбно, как для человека, не разбирающегося в маркетинге.
*   Не добавляй никаких комментариев или вступлений, только Markdown-структуру.
```

**Пользовательский промпт (запрос):**
```markdown
Проанализируй этот рассказ клиента и создай на его основе контент для лендинга в формате Markdown.

**Транскрипт интервью:**
{{user_interview_text}}

**Контент для лендинга:**
```

**Пример ожидаемого вывода от ИИ:**

```markdown
## Главный экран
**Заголовок:** Создаем продающие сайты для экспертов, даже если вы ничего не понимаете в дизайне
**Подзаголовок:** Пройдите короткий опрос, и наш ИИ-помощник соберет для вас лендинг, который начнет приносить заявки уже завтра.
**Кнопка:** Создать мой сайт

## Ваш результат (Точка Б)
**Заголовок:** Вы получите не просто сайт, а работающий инструмент
- **Готовый лендинг:** Выглядит профессионально и вызывает доверие.
- **Поток клиентов:** Сайт оптимизирован, чтобы превращать посетителей в покупателей.
- **Спокойствие и уверенность:** Вам больше не нужно думать о технических деталях — можно сфокусироваться на своем деле.

## Ваша проблема (Точка А)
**Заголовок:** Устали от конструкторов и фрилансеров?
Вы эксперт в своем деле, но попытки сделать сайт превращаются в кошмар: сложно, долго, дорого, а результат все равно не радует. Знакомо?

## Как это работает
**Заголовок:** Всего 3 простых шага к вашему сайту
1.  **Проходите интервью:** Отвечаете на несколько простых вопросов о вашем бизнесе.
2.  **ИИ анализирует:** Наш алгоритм изучает ваши ответы и проектирует контент.
3.  **Получаете сайт:** Через 5 минут у вас есть готовый лендинг. Останется только выбрать цвета!

## Почему мы, а не другие? (Увольнение конкурентов)
**Заголовок:** Хватит выбирать между «сложно» и «дорого»
- **Проще, чем конструктор:** Не нужно двигать блоки и разбираться в настройках.
- **Дешевле, чем студия:** Получите результат студийного качества по цене чашки кофе.
- **Умнее, чем фрилансер:** Наш ИИ использует проверенную маркетинговую методологию, а не просто рисует картинки.

## Призыв к действию
**Заголовок:** Готовы получить свой сайт?
Нажмите на кнопку, чтобы начать короткое интервью и получить прототип вашего будущего лендинга уже через 5 минут.
**Кнопка:** Начать и получить сайт
```

## Этап 3: Вопросы о визуальном стиле

После генерации контента можно задать несколько простых вопросов для кастомизации.

**Вопрос 1:**
```
Отлично, тексты готовы! Теперь давайте определимся со стилем. Какое настроение должен передавать ваш сайт? Выберите один или два варианта:

- Строгий и деловой
- Минималистичный и современный
- Дружелюбный и уютный
- Яркий и креативный
```

**Вопрос 2:**
```
Принято! А есть ли цвета, которые вам особенно нравятся или которые ассоциируются с вашим брендом? Можете просто назвать их или описать (например, «спокойные морские оттенки»).
```

*Эти ответы можно передать в CSS-генератор или использовать для выбора из готовых тем оформления.*
